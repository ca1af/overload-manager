# 와이어프레임 설명

> 설계 원칙: 모바일 퍼스트(Mobile First) 반응형 웹, 운동 중 한 손 조작 가능한 대형 터치 타겟, 최소 입력 횟수로 세트 기록 완료

---

## 1. 로그인/회원가입 화면

### 레이아웃 구조

```
┌─────────────────────────────┐  (max-width: 480px, 중앙 정렬)
│                             │
│    [로고: Overload Manager] │  ← 상단 로고 (SVG 바벨 아이콘 + 텍스트)
│                             │
│  ┌───────────────────────┐  │
│  │   이메일              │  │  ← Input (type=email, 44px height)
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │   비밀번호  [보기]    │  │  ← Input (type=password) + 토글 아이콘
│  └───────────────────────┘  │
│                             │
│  [         로그인          ]│  ← Primary CTA 버튼 (full width, 52px)
│                             │
│  [Google로 로그인]          │  ← 소셜 로그인 (향후 지원, 비활성 처리)
│                             │
│  계정이 없으신가요? [회원가입]│  ← 텍스트 링크
│  [비밀번호 찾기]            │
└─────────────────────────────┘
```

### 회원가입 폼 (동일 화면, 탭 전환)

```
┌─────────────────────────────┐
│  [로그인] [회원가입]        │  ← 탭 네비게이션
│  ─────────                  │  ← 활성 탭 언더라인
│                             │
│  ┌───────────────────────┐  │
│  │   닉네임              │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │   이메일              │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │   비밀번호            │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │   비밀번호 확인       │  │
│  └───────────────────────┘  │
│                             │
│  무게 단위: ● kg  ○ lb      │  ← 라디오 버튼 (기본: kg)
│                             │
│  [       회원가입          ]│
└─────────────────────────────┘
```

### 주요 UI 컴포넌트

- **입력 필드**: 44px 높이, 포커스 시 primary 색상 테두리 강조
- **CTA 버튼**: 52px 높이, 배경 primary 색상 (#4F46E5 인디고), 모서리 12px
- **에러 메시지**: 입력 필드 하단, 빨간색 소형 텍스트 (inline validation)
- **로딩 스피너**: 로그인 버튼 내부, 제출 중 표시

### 사용자 인터랙션 흐름

1. 이메일/비밀번호 입력 → Enter 키 또는 로그인 버튼으로 제출
2. 실패 시 인라인 에러 표시 (쉐이크 애니메이션)
3. 성공 시 대시보드로 슬라이드 전환
4. 회원가입 완료 시 이메일 인증 안내 모달 표시 후 로그인 탭으로 전환

### 주요 UI 컴포넌트

- **헤더**: 고정(sticky), 로고 + 알림 아이콘 + 아바타
- **요약 카드**: 가로 3분할 통계 수치, 배경 그라데이션 (인디고 계열)
- **오늘 운동 시작 버튼**: 그라데이션 배경, 아이콘+텍스트, 탭 뷰 최상단 고정
- **세션 카드**: 그림자, 모서리 16px, 탭으로 상세 진입
- **하단 탭 바**: 4개 탭, 활성 탭 인디고 색상 + 텍스트 표시

### 사용자 인터랙션 흐름

- 세션 카드 탭 → 세션 상세 화면
- "오늘 운동 시작" → 새 세션 생성 플로우
- 하단 탭 전환은 슬라이드 애니메이션 없이 즉시 전환 (운동 중 빠른 전환)
- Pull-to-Refresh로 최신 데이터 갱신

### 데이터 표시 방식

- 이번 주 요약: 월요일~일요일 기준, 실시간 업데이트
- 최근 세션: 최대 5개 표시, 페이지네이션 없이 "더 보기" 링크

## 3. 운동 선택

### 레이아웃 구조

```
┌─────────────────────────────────────┐
│ [<]   운동 추가              [완료(3)]│  ← 헤더 (뒤로가기 + 선택 수 표시)
├─────────────────────────────────────┤
│  ┌───────────────────────────────┐  │
│  │ 🔍  운동 검색...              │  │  ← 검색창 (autofocus)
│  └───────────────────────────────┘  │
│                                     │
│  전체  가슴  등  하체  어깨  팔  코어│  ← 가로 스크롤 카테고리 필터 탭
│  ────                               │  ← 활성 탭 언더라인
│                                     │
│  복합 운동                          │  ← 섹션 그룹 헤더 (sticky)
│  ┌───────────────────────────────┐  │
│  │ ○ 벤치 프레스                  │  ← 운동 아이템 (라디오 대신 체크박스)
│  │   가슴 · 복합 · 바벨          │  │  ← 서브텍스트 (카테고리·타입·장비)
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │ ✓ 인클라인 덤벨 프레스        │  ← 선택된 항목 (체크마크 + 강조색 배경)
│  │   가슴 · 복합 · 덤벨          │  │
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │ ○ 덤벨 플라이                 │  │
│  │   가슴 · 고립 · 덤벨          │  │
│  └───────────────────────────────┘  │
│                                     │
│  고립 운동                          │
│  ┌───────────────────────────────┐  │
│  │ ○ 케이블 플라이               │  │
│  └───────────────────────────────┘  │
│                                     │
│  [+ 커스텀 운동 추가]               │  ← P1 기능 진입점
│                                     │
│  ┌───────────────────────────────┐  │
│  │    선택된 운동 (3): 인클라인.. │  │  ← 하단 선택 요약 바
│  │    [         추가하기        ] │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 주요 UI 컴포넌트

- **검색창**: 화면 진입 시 키보드 자동 활성화, 실시간 필터링
- **카테고리 탭**: 가로 스크롤, 활성 탭 인디고 강조, 배지로 개수 표시
- **운동 아이템**: 48px 높이, 좌측 체크박스, 운동명 + 서브텍스트
- **선택 운동 요약 바**: 화면 하단 고정, 선택 수 표시 + 추가 CTA
- **운동 상세 바텀시트**: 운동 아이템 길게 누르면 주동근/권장 반복수 정보 표시

### 사용자 인터랙션 흐름

1. 카테고리 탭 선택 또는 검색어 입력 → 목록 실시간 필터
2. 운동 탭 → 토글 선택/해제 (체크박스)
3. 운동 길게 누르기(long press) → 운동 정보 바텀시트
4. 하단 "추가하기" 탭 → 세션 화면으로 이동, 선택 운동 추가됨

## 4. 과부하 설정 (이전 기록 참고 패널)

> 별도 독립 화면이 아닌, 세트 기록 화면 내 슬라이드 업 패널(Bottom Sheet) 형태

```

│ │           ━━━ (드래그 핸들)     │  │
│ │                                 │  │
│ │  벤치 프레스 과부하 현황         │  │
│ │                                 │  │
│ │  직전 세션 (3/12):              │  │
│ │  ┌─────────────────────────┐   │  │
│ │  │ Set1  80kg × 8회   ✓   │   │  │
│ │  │ Set2  80kg × 8회   ✓   │   │  │
│ │  │ Set3  80kg × 7회   ✓   │   │  │
│ │  │ 총 볼륨: 1,840 kg        │   │  │
│ │  └─────────────────────────┘   │  │
│ │                                 │  │
│ │  추정 1RM: 106 kg               │  │
│ │                                 │  │
│ │  오늘 목표 제안                  │  │
│ │  ┌─────────────────────────┐   │  │
│ │  │ A  82.5kg × 8회  (중량↑)│   │  ← 선택 가능한 카드
│ │  └─────────────────────────┘   │  │
│ │  ┌─────────────────────────┐   │  │
│ │  │ B  80kg × 9회  (반복↑) │   │  │
│ │  └─────────────────────────┘   │  │
│ │                                 │  │
│ │  최근 추이 (최대 중량, 8주)      │  │
│ │  ╔═══════════════════════╗     │  │
│ │  ║  82.5                 ║     │  │
│ │  ║          /            ║     │  │  ← 미니 라인 차트
│ │  ║    75  80             ║     │  │
│ │  ╚═══════════════════════╝     │  │
│ │                                 │  │
│ │  [ 닫기 ]                       │  │
│ └─────────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 주요 UI 컴포넌트

- **바텀시트**: 드래그 핸들, 반투명 오버레이 배경, 최대 화면 80% 높이
- **직전 세션 기록 테이블**: 세트별 무게·반복수·완료 여부
- **목표 제안 카드**: 탭으로 해당 무게/반복수 자동 채우기 (편의 기능)
- **미니 추이 차트**: 8주 최대 중량 라인 차트, Recharts 기반

### 사용자 인터랙션 흐름

1. 세트 기록 화면에서 "이전 기록" 버튼 탭 → 바텀시트 슬라이드 업
2. 목표 제안 카드 탭 → 현재 세션 입력 필드에 값 자동 채움
3. 드래그 다운 또는 "닫기" 탭 → 바텀시트 닫힘
4. 화면 반 이상 스크롤 시 전체 화면 확장

### 주요 UI 컴포넌트

- **세트 행**: 세트 번호 + 무게 입력 + 반복수 입력 + 완료 체크박스 (좌→우 레이아웃)
- **숫자 입력 필드**: 탭 시 숫자 키패드 자동 활성, 이전 세션 값 placeholder로 표시
- **완료 체크박스**: 48×48px 대형 타겟, 체크 시 녹색 + 해당 행 배경 연녹색
- **세트 추가 버튼**: 점선 테두리, 하단에 항상 표시
- **볼륨 실시간 계산**: 완료된 세트만 합산 표시
- **휴식 타이머**: 원형 진행 바 + 대형 숫자, 반투명 오버레이

### 사용자 인터랙션 흐름

1. 무게 입력 → 반복수 입력 → 완료 체크 (순차 진행, Tab 키 지원)
2. 완료 체크 시 휴식 타이머 오버레이 자동 팝업
3. 타이머 종료 or "건너뜀" → 다음 세트 무게 입력 포커스 이동
4. 세트 행 스와이프(좌) → 삭제 버튼 노출 (편집/삭제)
5. 세트 행 길게 누르기 → 드래그 앤 드롭으로 순서 변경

### 데이터 표시 방식

- 이전 세션 값: placeholder로 연회색 표시, 입력 시 사라짐
- 자동 저장: 무게/반복수 입력 직후 debounce 500ms 후 자동 저장 (실패 시 로컬 캐시)
- 실시간 볼륨 계산: 완료 체크된 세트만 포함

## 선정 근거

### React 18 + TypeScript

- **백엔드(Spring Boot + Kotlin)와 팀원 기술 스택 연속성**: Kotlin의 타입 안정성과 유사한 개발 경험 제공
- **생태계 성숙도**: 운동 기록 앱에 필요한 복잡한 폼, 실시간 UI 갱신, 차트 라이브러리 모두 React 생태계에서 검증됨
- **P2 모바일 앱 전환 용이**: React Native로의 코드 공유 가능 (비즈니스 로직, 타입, API 레이어)
- **Concurrent Features**: 세트 기록 중 타이머 + 입력 동시 처리에서 반응성 향상

### TanStack Query + Zustand 조합

- **TanStack Query**: 서버 상태(운동 목록, 세션 기록, 통계)의 캐싱·동기화·낙관적 업데이트를 선언적으로 처리
  - 세트 완료 시 낙관적 업데이트로 즉각 UI 반응 → 운동 중 UX 지연 제거
  - 네트워크 에러 시 자동 재시도 + stale-while-revalidate
- **Zustand**: 클라이언트 전용 상태(휴식 타이머 카운트다운, 진행 중 세션 임시 데이터) 관리
  - Redux 대비 보일러플레이트 최소화, 운동 중 타이머처럼 고빈도 업데이트에 적합
  - Redux를 택하지 않은 이유: 미들웨어·액션·리듀서 구조가 이 규모 앱에 과도하게 복잡

### shadcn/ui + Tailwind CSS

- **shadcn/ui**: Radix UI 기반의 접근성(A11y) 완비 컴포넌트 (Dialog, Select, Checkbox 등)
  - 바텀시트, 모달, 드롭다운 등 과부하 설정 화면에 필요한 복잡한 UI를 즉시 활용
  - 소스 코드 복사 방식으로 완전한 커스터마이징 가능 (운동 앱 특화 디자인)
- **Tailwind CSS**: 유틸리티 클래스로 반응형 레이아웃 빠른 구현, 모바일 퍼스트 breakpoint 처리 직관적

### Recharts

- React 컴포넌트 기반으로 TypeScript 타입 완비
- 최대 중량 추이(LineChart), 주간 볼륨(BarChart) 모두 지원
- SVG 기반으로 반응형 처리 간단 (`<ResponsiveContainer>`)
- D3 대비 학습 곡선 낮음, Chart.js 대비 React 통합 자연스러움

### Vite

- CRA(Create React App) 대비 개발 서버 콜드 스타트 10~100배 빠름
- HMR(Hot Module Replacement)이 빠르므로 운동 기록 UI 반복 개발에 유리
- Spring Boot 백엔드와 프록시 설정 간단 (`vite.config.ts`의 `server.proxy`)

### React Hook Form + Zod

- 세트 기록 입력(무게: 양수, 반복수: 정수)과 같은 유효성 검사를 스키마로 선언
- 불필요한 리렌더링 최소화 (운동 중 입력 필드 반응성 유지)
- Zod 스키마를 백엔드 DTO와 공유 가능 (모노레포 구조로 확장 시)

### Framer Motion + vaul

- **Framer Motion**: OverloadInfoSheet, SessionSummaryModal에만 lazy import 적용하여 번들 크기 최소화
  - 페이지 전환(하단 탭 등)은 CSS transition으로 대체 — 운동 중 화면 전환 속도 우선
- **vaul**: shadcn/ui 공식 권장 바텀시트 라이브러리 (Radix 기반)
  - 드래그 핸들, 스냅 포인트, 80% 높이 제한, 드래그 다운 닫기를 선언적으로 구현
  - OverloadInfoSheet의 핵심 인터랙션을 네이티브 수준 품질로 제공

### P2 모바일 앱 확장 고려

- **React Native + Expo**: 현재 React + TypeScript 코드베이스에서 비즈니스 로직, API 훅(TanStack Query), Zod 스키마 그대로 재사용 가능
- **PWA**: vite-plugin-pwa를 초기 구조에 포함하되 서비스 워커는 비활성화 상태로 시작, P2에서 오프라인 캐시 전략 추가 시 코드 변경 최소화
- UI 컴포넌트만 React Native Paper / NativeWind로 교체하면 코드 공유율 60~70% 달성 가능
